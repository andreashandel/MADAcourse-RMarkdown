2019-10-26

Overview

This module provides a few motivating examples of real world data analyses that I hope you will be able to do once you complete the course. We‚Äôll get into the details of what data analysis is and how to do it very soon. But to start with, I want to provide a few hopefully inspiring and motivating examples of data analysis and use. You‚Äôll be doing similar projects yourself soon! In fact, you will be fully reproducing several analyses below.

Learning Objectives

  • Get motivated toward learning and doing data science.
  • See what‚Äôs possible with R and what you‚Äôll be able to do yourself soon.

Motivating Videos

There are lots of good videos online discussing different aspects of the vast topic of data analysis. TED is an especially good source for such videos.

TED talk 1

Arguably one of the best communicators of public health data was the late Hans Rosling. He has several great entertaining and enlightening TED talks. Here is one of them - a statistics talk with >13 million views!

You can watch more TED talks by Hans Rosling here. The software he uses to illustrate data is called Gapminder and available online on the Gapminder website.

TED talk 2

In this entertaining and empowering talk, Talithia Williams argues for the utility of careful assessment of data in your personal life.

TED talk 3

In this talk, Sebastian Wernicke gives some cautionary suggestions about trusting data too much and argues for critical assessment as an important part of any data analysis and decision process.

More TED talks

The TED website has a large number of data related talks. Feel free to explore as much as you like. If you find especially good ones, post them on the class discussion board.

Case studies

It is by now very common that individuals post data analyses online in a reproducible manner, i.e., providing all the code and data to recreate it.

TidyTuesday

An interesting initiative is TidyTuesdays, where every Tuesday a new dataset is released here and individuals are encouraged to analyze it. Later in this course, you will be participating in some TidyTuesdays yourself.

Individuals who participate in TidyTuesday often post their projects online and link to it on social media. We will look at a nice example by David Robinson analyzing a wine rating dataset. He does a live recording of him digging into the data.

You don‚Äôt need to watch the whole 85 minutes, but do watch at least the first 15 minutes to see him coding/analyzing in real-time. You might just get hooked and keep watching üòÑ. It should be interesting even if you don‚Äôt care about wine. If you have no R experience, you won‚Äôt understand the details, but you will still get the overall idea of how he does the analysis. And his thinking out loud is informative as well. If you like the ability to look over his shoulder while he does data analysis, he has a bunch more of those screencasts.

Dave also does his work reproducibly and shares all the code. See the links provided in the video description. For convenience, here is the link to download the code for his analysis. Just save the file after it opens in your browser. It should be named wine-ratings.Rmd. Click on it, and it should open in RStudio (if you don‚Äôt have R/RStudio installed yet, go through module 2, then come back).

Once the file is open in RStudio, you might see at the top of the file a message suggesting that you need to install several packages to run the file. Do so. If you don‚Äôt get that auto-suggestion, you need to install the packages by hand. Every package that is called with the library() command needs to be installed first. There is one package that is only available for Linux (and maybe Mac?) computers. So we need to alter the code a bit to make things work for everyone. Find these lines of code:

library(doMC)
registerDoMC(cores = 4)

and add a # in front of them to turn them into comments and therefore disable them (you could also delete these lines, but then they are gone for good, so disabling with # is often better).

Once you have done that and installed all other packages, hit the Knit button. If you are still missing a package, you will get an error message. If things work out, the whole script will run a while (depending on the speed of your computer), and you should get a document/report which reproduces the complete analysis he did in the video!

Again, most or all of the R code might not make sense to you just yet. That‚Äôs ok. The main point is to show you a nice example of a data analysis done in a way that others (you) can reproduce it with just a few clicks.

Note the name of the last wine reviewer in the final figure of the knitted report.

A simple Kaggle analysis

This is a very simplified example of a ‚Äúfull‚Äù analysis I wrote. It is only for illustrative purposes, the whole analysis is rather incomplete and poor quality üòÑ, but hopefully still a bit helpful in getting an overall feel how such analyses look.

The analysis is based on and inspired by the Kaggle Titanic challenge. The RMarkdown file gives some more details. This is the link to the Github repository which contains all files you need to look at and run the analysis yourself. In the repository, you will find one Rmd file and 1 csv files. Download them (right-click and save as). The double-click the Rmd file, which should open in RStudio. You will need to install a few R packages first. Also make sure you are in the correct folder by doing Session -> Set Working Directory -> To Source File Location. Then knit the document, which should run the complete analysis, produce a report which should open in the viewer, and save a file to the folder which you could upload to Kaggle.

Take a close look or save the decision tree.

Research Project

Fully open and reproducible analyses are also becoming common in academic research, with authors publishing everything needed to reproduce their paper and results. In my research group, we try to adhere to this approach as much as possible. As an example, I‚Äôm sharing a recent project/paper by Brian McKay (one of the grad students in my group). This is a full analysis project with the main product being a peer-reviewed journal article (the paper is currently under review). As you will see, it is more complex and complete compared to the examples above. You will also see how easy it is to ‚Äúwrite a whole paper‚Äù with just a few clicks of a button.

This is the link to the Github repository which contains all files of this project/paper. You need to have joined the epid8060fall2019 organization for you to be able to access this link. If you haven‚Äôt done yet, go through the Github materials to figure out how. The repository contains brief instructions describing how you can reproduce all of his analyses and reproduce his complete manuscript. As you will see, it is simply a combination of R scripts and Rmarkdown files (in this case a version called bookdown), which allow a fully automated generation of a complete data analysis and manuscript, including all figures, tables, references, etc. This is an incredibly efficient workflow and one you‚Äôll learn and use in this class.

Note the number of files in the Tables folder after you run all analysis scripts.
